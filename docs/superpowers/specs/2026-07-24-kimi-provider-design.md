# TasteLab AI Kimi Provider Design / TasteLab AI Kimi 供应商设计

## Decision / 决策

TasteLab AI uses Kimi for text and image tasks. Live web search is disabled for localization, and every market conclusion is labeled as an AI inference that requires human validation.

TasteLab AI 使用 Kimi 承担文本与图片任务。本地化阶段关闭实时联网搜索，所有市场结论均标记为 AI 推断并要求人工验证。

## Runtime / 运行方式

- The browser continues to call only `POST /api/ai`.
  浏览器继续只调用 `POST /api/ai`。
- Vercel Functions call Moonshot's OpenAI-compatible Chat Completions endpoint.
  Vercel Functions 调用 Moonshot 的 OpenAI 兼容 Chat Completions 接口。
- Kimi JSON Mode produces one JSON object, which must pass the existing task-specific Zod schema before the result is accepted.
  Kimi JSON Mode 生成单个 JSON 对象，结果必须通过现有任务级 Zod Schema 校验后才能被接收。
- The `openai` npm package remains only as a compatible HTTP client; its Base URL points to Moonshot.
  `openai` npm 包仅作为兼容协议 HTTP 客户端保留，其 Base URL 指向 Moonshot。

## Capability Rules / 能力规则

- Brief analysis, recipe standardization, robot-data generation, final-recipe generation, and translation use text input.
  Brief 解析、菜谱标准化、机器人数据生成、最终菜谱生成和翻译使用文本输入。
- Experiment comparison may include up to three existing compressed image Data URLs.
  实验比较可包含最多三张现有压缩图片 Data URL。
- Localization never returns citations or sourced facts while search is disabled.
  联网搜索关闭期间，本地化不返回引用或来源事实。
- Any sourced fact accidentally generated by the model is converted into an AI inference and its URL is discarded.
  模型意外生成的来源事实会被转换为 AI 推断，其 URL 会被丢弃。

## Configuration / 配置

```dotenv
AI_PROVIDER=moonshot
MOONSHOT_API_KEY=your_kimi_api_key
MOONSHOT_MODEL=kimi-k3
DEMO_ACCESS_CODE=your_shared_demo_code
SESSION_SECRET=use_a_random_secret_of_at_least_32_characters
```

These values exist only in server-side environment variables and must not be committed or exposed to the browser.

这些变量只存在于服务端环境中，不得提交到仓库或暴露给浏览器。

## Acceptance Criteria / 验收标准

- Missing Kimi configuration returns `503 AI_NOT_CONFIGURED`.
  缺少 Kimi 配置时返回 `503 AI_NOT_CONFIGURED`。
- Invalid JSON or a schema mismatch returns `422 AI_INVALID_OUTPUT` and does not unlock the next stage.
  JSON 非法或 Schema 不匹配时返回 `422 AI_INVALID_OUTPUT`，且不解锁下一阶段。
- Experiment photos are sent as Kimi-compatible `image_url` content parts.
  实验照片以 Kimi 兼容的 `image_url` 内容块发送。
- Localization always displays the inference-only warning and never displays source links.
  本地化始终显示纯推断提示，且不展示来源链接。
- No request uses `OPENAI_API_KEY` or OpenAI API quota.
  任何请求均不使用 `OPENAI_API_KEY` 或 OpenAI API 额度。
